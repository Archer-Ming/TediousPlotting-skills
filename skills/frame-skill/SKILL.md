## ⚠️重要规则！！!

- 一定确保准确理解用户的需求，识别其构思的想法布局。
- 参考模板提供了多种架构图模板风格，供你学习，你要做的是去学习其图形风格以及方框搭配组合。按用户需求进行绘制，排版。
- 你不必局限于我的参考模板，不要过于固化，可以在学习后自由发挥板块方框的搭配，只要确保最终的架构图符合美学，呈现的逻辑清晰。
- ****输出的架构图如果是图片PNG，需清晰度dpi = 400 达到期刊水准**。**
- 根据用户输入语言情况灵活选择语言，如果用户的输入主要是中文 ，使用中英文混合标签，英文用于技术术语。如果用户的输入主要是英文，全程使用英文标签。
- **Times New Roman（用于英文/数学），Noto Sans CJK SC + SimHei（用于中文）。含中文的图，`font-family` 必须把 CJK 字体写在最前，否则中文会丢失。**
- SVG 是唯一真相来源。PNG 和 PPTX 通过 `render_helpers.py` 从 SVG 生成。**绝不**手动编辑 PNG/PPTX，始终编辑SVG并重新渲染。
- 参考模板的架构图中，请忽略其语言文字，数据，逻辑。你要学习的是图形，图框配色组合，线框搭配，整体布局的美感。
- ★ ✓ ✗ ① ↻ 等特殊字符可能渲染成豆腐框，需要这些图形时一律用 SVG 图元绘制（见 `reference/drawing_recipes.md`）。

## 工作流程

```
1. 识别用户需求的图类型，仔细剖析用户的架构布局逻辑，确定是references/styles.md中的哪一种布局
2. 读取references中对应类型的SVG参考文件，学习其逻辑架构，你可以自由组合多个文件样式
3. 阅读palettes.md并选取合适调色板风格；阅读reference/drawing_recipes.md
   获取坐标计算与排版的可执行规则。
4. 编写和绘制SVG，并根据用户需求，使用render_helpers.py对其进行渲染为PNG/PPTX
5. 确保遵守规则和自检项,如果不符需重新绘制
```

## 输入格式

| 语言                 | 类型     | 处理方式                                              |
| -------------------- | -------- | ----------------------------------------------------- |
| 自由散文 / 对话      | 自然语言 | 提取实体+关系 → 回显结构化形式 → 等用户确认 → 渲染 |
| 输入有明确的编号步骤 | 结构化   | 按模板映射字段完成绘制并渲染                          |

**对于自然输入，始终先回显解析结果给用户，等待用户确认再渲染。确保流程图质量绝对不能跳过！！**

## 自检

- [ ] **架构图任何分支都不能悬空。** 所有路径必须终止或汇合。
- [ ] 每个容器的标题不与它们重叠。没有文本跨越形状边界或其他文本元素。所有箭头都有可见的箭头，并清晰地从源指向目标。虚线（跳转/反馈）与实线在视觉上可区分。
- [ ] 架构图涉及到的文字符号不能有任何乱码和豆腐框。
- [ ] 整个架构图需大致呈现同一种风格,模块之间配色，布局大小，平衡感合理。
- [ ] 架构图各板块方框组合后整体大小要适配，不能出现局部较大或整体失衡。
- [ ] 所有元素都在viewBox内,无越界。
- [ ] 已运行 `python render_helpers.py validate <svg>`,无problem。

**如果以上任何一条不满足，请重新检查，改进，渲染！！！**
