name: Frame Skill
description: 本技能是通过自然语言描述或结构化清单生成期刊论文架构图。当用户提到架构图或要求"画"/"可视化"/"画图"任何流程架构步骤、结构布局、层级关系时，使用这个skill。即使用户只是随口描述内容，也要确保输出的架构图逻辑准确，风格符合美学。输出 SVG、PPTX。
---
## 工作流程

```
1. 识别用户需求的图类型和输入格式（自然语言 / 结构化）
2. 仔细剖析用户的架构布局逻辑
3. 阅读学习参考模板并选取合适风格
4. 绘制并渲染架构图
5. 确保遵守规则和自检项，完成后输出SVG或PPTX
```

## 重要规则！！!

- 一定确保准确理解用户的需求，识别其构思的想法布局。
- 参考模板提供了多种架构图模板风格，供你学习，你要做的是去学习其图形风格，按用户需求进行绘制，排版。
- 参考模板每一种风格模块里都包含多种板块，你可以学习后自主选择不同模块的板块方框进行搭配组合。
- 你最后只能根据用户风格（如果没有提及则自主决策一个风格组合），去组合各板块的图形并结合用户想法做成架构图。
- 你不必局限于我的参考模板，不要过于固化，可以在学习后自由发挥板块方框的配色，搭配，只要确保最终的架构图符合美学，呈现的逻辑清晰。
- 架构图涉及的字体必须为 **Times New Roman。**

## 参考模板

⚠️**参考模板的架构图中，请忽略其语言文字，架构逻辑，你要学习的是图形，图框配色组合，线框搭配，整体布局的美感。**

| 类型   | 参考模板                        | 何时打开                                  |
| ------ | ------------------------------- | ----------------------------------------- |
| 框架图 | [assets](assets\visual_style.pptx) | 当涉及架构/步骤/决策/工作流/流程/逻辑布局 |

如果不明确，**询问用户，不要自主决策**。

## 输入格式

| 信号                                                        | 类型     | 处理方式                                                  |
| ----------------------------------------------------------- | -------- | --------------------------------------------------------- |
| 自由散文 / 对话                                             | 自然语言 | 提取实体+关系 → 回显结构化形式 → 等用户确认"ok" → 渲染 |
| 输入有明确的 `节点:` / `nodes:` / `A -> B` / 编号步骤 | 结构化   | 按模板映射字段，渲染                                      |

**对于自然输入，始终先回显解析结果再渲染。确保流程图质量绝对不能跳过！！**

## 输出格式

按用户需求输出以SVG（默认）或 PPTX, PPTX需将流程图呈现于正中央。

## 自检

- [ ] **架构图任何分支都不能悬空。** 所有路径必须终止或汇合。
- [ ] 架构图涉及到的文字符号不能有任何乱码和错乱，且文字与文字，文字与图框不得出现重合。
- [ ] 整个架构图需大致呈现同一种风格,模块之间配色，布局大小，平衡感合理。
- [ ] 架构图各板块方框组合后整体大小要适配，不能出现局部较大或整体失衡。

**如果以上任何一条不满足，请重新改进绘制！！！**
