# 绘制配方 Drawing Recipes

读完 `styles.md` 选定布局后再读本文。这里给的是**可直接套用的数值规则**,专门用来避免两个失败:文字重叠、布局失衡。

---

## 画布与坐标系

| 布局类型             | viewBox            | 说明                               |
| -------------------- | ------------------ | ---------------------------------- |
| 水平流程 / 双轨      | `1600 x 600~800` | 宽 > 高,留出箭头与标签空间         |
| 分层网络 / 树 / 泳道 | `1600 x 900`     | 接近 16:9,适配 PPTX 幻灯片         |
| 模块系统 / 循环      | `1400 x 1000`    | 偏方形,容纳上下分层或环形布局      |
| 放射状               | `1200 x 1100`    | 接近正方形,卫星节点四面展开        |
| 网格矩阵             | `1500 x 900`     | 视行列数微调:每多一列宽 +220       |
| 时间轴               | `1600 x 750`     | 扁宽,主轴横贯,上下各留一排卡片高度 |

- viewBox 一旦定下,所有元素必须落在内部(`render_helpers.py` 的验证器会检查越界)。
- 内容四周留至少 40px 边距;标题区通常占顶部 90~110px。

## 容器尺寸与间距

- 容器圆角:`rx=12~14`。模块圆角:`rx=6~8`。
- **容器之间**水平/垂直间距 ≥ 50px(留给连接线和箭头)。
- **模块之间**(同一容器内)间距 8~14px。
- 容器标题字号 18~26pt 粗体;模块主标签 14~16pt;模块副标签 11~13pt;注释 11~13pt。
- 同一张图里,**同层级的容器尽量等高**,同类模块尽量等宽——失衡感大多来自尺寸随意。

## 文字排版

1. **一行文字的预估宽度** ≈ 字符数 × 字号 × 0.6(中文按 1.0,英文按 0.55)。
   若预估宽度 > 容器宽度 − 24,必须换行或缩小字号。
2. **换行**:SVG 的 `<text>` 不自动换行。用多个 `<text>` 元素,每行 y 递增
   `字号 × 1.4`。或用 `<tspan dy="...">`。
3. **垂直堆叠**:容器内文字块,首行 y = 容器顶 + 32,之后每行 +20~24。
   底部注释 y = 容器底 − 14。
4. **居中**:`text-anchor="middle"`,x = 容器中心。左对齐标题 x = 容器左 + 20。
5. 文字**绝不**跨越形状边界或压住另一段文字。画完在脑中按上面的宽度公式复核。
6. 数学符号(θ、∇、σ²、⟨⟩、≤ 等)可直接用 Unicode,CJK/serif 字体都支持;
   但 ★ ✓ ✗ ① ↻ 这类符号在渲染器里可能变豆腐框,**一律用 SVG 图元画**。

## 连接线与箭头

- 先在 `<defs>` 定义 `marker`,实线箭头 `#6B7280`,虚线箭头 `#9CA3AF`。
  `marker-end` 引用的 id 必须存在,否则验证器报错。
- 连接线起点/终点贴在**容器边缘**,不要从容器中心发出(会被容器盖住)。
- 正交连接(树、泳道、分层)只走水平段和垂直段,转角为直角,不画斜线。
- 曲线连接(反馈、放射)用二次贝塞尔 `Q`,弧度平缓即可。
- 同类连接线样式必须一致;实线 = 主数据流,虚线 = 跳跃/反馈/共享。

## 各布局的专属坐标提示

**01 分层网络**:容器横向排开,容器内模块按行码放;阶段间用粗一点的
连接线(2.5px);残差用虚线 `path` 绕过主路径。

**02 水平流程**:每个模块 200~220 宽、150~160 高,模块间隔 60~70(放箭头+微标签)。
步骤序号用左上角 `<circle>` + 数字。

**03 模块系统**:按"层带"组织(UI 带 / 核心带 / 存储带),每带一个浅色长条容器,
带内模块等宽均分;跨带连接线垂直贯穿。

**04 层级树**:深度决定颜色(根=lavender,中层=blue,叶=mint)。
父底中点向下引线,到一条水平母线,再分别下引到各子节点顶部。全程正交。

**05 循环**:4 个节点放东南西北,中心放 hub 圆;闭环用 4 段 `Q` 曲线顺时针相连;
要留一个"退出条件"出口,避免分支悬空。

**06 双轨**:上下两条轨,各用不同色系长条容器;两轨模块 x 坐标尽量对齐,
对比才一目了然;交叉连接用虚线。

**07 网格矩阵**:行标签列固定宽(~240),数据列等宽(~215);
表头行单独一行高(~70),数据行等高(~92);获胜行/列用 accent 描边高亮。

**08 放射状**:hub 居中,卫星沿半径 R 的圆周等角分布(N 个则每 360/N 度一个）；
先画连接线再画节点(节点压住线头);卫星卡片尺寸全部一致。

**09 泳道**:左侧角色 gutter 固定宽(~170),泳道等高;步骤块垂直居中于所在泳道;
跨泳道换手用正交连接;可在顶部加阶段标尺。

**10 时间轴**:一条主轴横贯;里程碑节点等间距;卡片**上下交错**(奇数在上、偶数在下)
以免重叠;节点用实心圆 + 白描边。

## 配色

- 从 `palettes.md` 选**一个**调色板,全图只用它 + 共享 neutrals。
- 容器用 `fill`(浅),模块用 `solid`(饱和),边框用 `solid` 或其 80% 不透明度。
- 饱和块上的文字用白色;浅块上的文字用 `#1F2937`。
- 想突出某一个模块/行/列,用 accent 家族或加一圈 accent 描边,**只突出一个重点**。

## 自检

- [ ] 所有元素在 viewBox 内,无越界。
- [ ] 没有任何文字重叠、跨界、压线。
- [ ] 所有箭头都有可见箭头头,方向正确;虚线与实线视觉可区分。
- [ ] 含中文时 `font-family` 是 CJK 优先,无豆腐框。
- [ ] 全图单一调色板,同层级尺寸均衡,无局部过大。
- [ ] 没有悬空分支:每条路径都终止或汇合。
- [ ] 用 `render_helpers.py validate` 跑一遍,0 problem 才算过。
