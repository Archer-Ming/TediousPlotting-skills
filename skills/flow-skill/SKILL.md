---
## 工作流程

```
1. 识别用户需求的图类型和输入格式（自然语言 / 结构化）
2. 仔细剖析用户的流程逻辑
3. 阅读学习参考模板并选取合适风格
4. 绘制并渲染流程图
5. 确保遵守规则和自检项，完成后输出SVG或PPTX
```

## ⚠️重要规则！！

- 一定确保准确理解用户的需求，识别其构思的想法布局。
- 参考模板提供了流程图板块化中每一部分的风格，供你学习，你要做的是去学习其图形风格，按用户需求进行绘制流程排版。
- 参考模板每一种风格里的模块都提供了多个可选项，你学习后自主选择搭配。
- 你最后只能根据用户风格（如果没有提及则自主决策一个风格组合），去组合各板块的图形并结合用户想法做成流程图。
- 你不必局限于我的参考模板，不要过于固化，可以在学习后自主发挥板块的配色，搭配，只要确保最终的流程图符合美学，呈现的逻辑清晰。
- 流程图涉及的字体必须为 **Times New Roman。**
- 流程图各模块组合后整体大小要适配，不能出现局部较大或整体失衡**。**

## 参考模板

| 类型            | 参考模板                 | 何时打开                                  |
| --------------- | ------------------------ | ----------------------------------------- |
| 流程图 / 框架图 | [assets\visual_style.pptx](assets\visual_style.pptx) | 当涉及架构/步骤/决策/工作流/流程/逻辑布局 |

如果不明确，**询问用户，不要自主决策**。

## 输入格式

| 信号                                                        | 类型     | 处理方式                                                  |
| ----------------------------------------------------------- | -------- | --------------------------------------------------------- |
| 自由散文 / 对话                                             | 自然语言 | 提取实体+关系 → 回显结构化形式 → 等用户确认"ok" → 渲染 |
| 输入有明确的 `节点:` / `nodes:` / `A -> B` / 编号步骤 | 结构化   | 按模板映射字段，渲染                                      |

**对于自然输入，始终先回显解析结果再渲染。确保流程图质量绝对不能跳过！！**

## 自检

- **流程图任何分支都不能悬空。** 所有路径必须终止或汇合。
- **单张图节点数上限 40。** 超过则建议用 `subgraph` 拆解或分多张图。
- 流程图涉及到的文字符号不能有任何乱码和错乱。
- 整个流程图需大致呈现同一种风格。

**如果以上任何一条不满足，请重新改进绘制！！！**
