## 输入层级

| 信号                                                              | 层级           | 处理方式                                                  |
| ----------------------------------------------------------------- | -------------- | --------------------------------------------------------- |
| 输入以 `flowchart`、`graph TD`、`mindmap`、`digraph` 开头 | L3（原始代码） | 跳过解析，直接渲染                                        |
| 输入有明确的 `节点:` / `nodes:` / `A -> B` / 编号步骤       | L2（结构化）   | 按模板映射字段，渲染                                      |
| 自由散文（"用户登录后..."）                                       | L1（自然语言） | 提取实体+关系 → 回显结构化形式 → 等用户确认"ok" → 渲染 |

**对于 L1 输入，始终先回显解析结果再渲染。** 示例：

- 节点：登录 → 验证邮箱 → [成功 → 主页] / [失败 → 重试]
- 决策点：验证邮箱
  确认无误的话我就开始画。

**这一步的确认是本 skill 最重要的质量控制点，绝对不能跳过。**

## 自检

- **流程图任何分支都不能悬空。** 所有路径必须终止或汇合。
- **单张图节点数上限 40。** 超过则建议用 `subgraph` 拆解或分多张图。
- **思维导图必须只有一个根节点。** 如果用户给了多个顶层概念，询问是合并到一个合成根节点下，还是生成多张图。
